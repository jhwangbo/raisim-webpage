Introduction¶

RaiSim is a physics engine developed by Jemin Hwangbo, Dongho Kang and other researchers at Robotic Systems Lab, ETH Zurich. It was designed to provide both the accuracy and speed for simulating robotic systems. However, it is a generic rigid-body simulator and can simulate any rigid body very efficiently.

Here is a list of advantages of using RaiSim:

  • The speed is benchmarked against other popular physics engine. ([1])

  • The accuracy of RaiSim has been demonstrated through a number of papers ([2], [3])

  • Easiest C++ simulation library to learn/use

  • A minimum number of dependencies (only on STL and Eigen)

System Requirements¶

RaiSim is supported only on the following systems

  • Linux only. support ubuntu 16.04 and 18.04 but might work on other distributions

  • g++, gcc > 6

  • cmake > 3.10

  • The binaries contain avx2 instructions. If your CPU is 7-year-old or older (before Haswell), it is likely that raisim does not work on your machine.

Example code¶

Here is an example of an RaiSim application code.

#include “raisim/World.hpp”

int main() {
  raisim::World world;
  auto anymal = world.addArticulatedSystem(PATH_TO_URDF);
  auto ball = world.addSphere(1, 1);
  auto ground = world.addGround();
  world.setTimeStep(0.002);
  world.integrate();
}

Here is a cmake file to compile above application code.

cmake_minimum_required(VERSION 3.10)
project(raisim_examples LANGUAGES CXX)

find_package(raisim CONFIG REQUIRED)
find_package(Eigen3 REQUIRED HINTS ${Eigen3_HINT})

include_directories (${EIGEN3_INCLUDE_DIRS})

add_executable(APP_NAME ${file_name})
target_link_libraries(APP_NAME PUBLIC raisim::raisim)
target_include_directories(APP_NAME PUBLIC ${CMAKE_CURRENT_SOURCE_DIR}/include)

Tips¶

The best reference to RaiSim is the header files. If you want to know how to change the gravity of the world, search with the keyword “gravity” in World.hpp. If you want to know how to apply external forces to an articulated system, search with the keyward “external” in ArticulatedSystem.hpp. Users who are familiar with robotics convention rarely need this document.
